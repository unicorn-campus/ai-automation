# 1회차 부정제출 검토서

## 검토 방법

3종 신호로 판정하며, 공통과제(요약·회의록·시장조사)는 동일 가이드를 따르므로 구조 유사성은 정상으로 간주(오탐 방지 최우선).

- **신호 A — 수강생 간 복제**: 과제별로 제출물 짝 비교. 섹션 헤더·공백·마크다운 이스케이프 제거 후 '고유 내용'만  
  대상으로 `difflib.SequenceMatcher` 비율 및 토큰 자카드 유사도 계산(재현 가능). 고유내용 유사도 0.85↑ = 복제 의심.
- **신호 B — 예제/보일러플레이트 복붙**: 강사 예제·AGENTS.md 기본 골격과 실질 동일하고 본인 업무 개인화가 없는 경우.
- **신호 C — 메타데이터 이상**: owner·createdTime·modifiedTime로 계정 공유·전달 복제·반복개선 흔적 부재 확인(보조 신호).

## 판정 결과: **부정제출 없음** (확정 0명 · 의심 0명)

### 신호 A (내용 유사도)

- 전 과제 쌍 비교 결과 **고유내용 유사도 0.85 이상 쌍 0건**.
- 0.70 이상은 단 1쌍: `신원석 ↔ 신원석`(본인의 시장조사 2건, PQC 시장분석 vs PUF 시장조사, 0.736)  
  → 동일인의 유사 주제 프롬프트로 **부정 아님**.
- 수강생 간 복제 정황 없음. 공통과제 구조 유사로 인한 오탐도 없음(정규화 정상 작동).

### 신호 B (예제/보일러플레이트)

- 제출물은 각자 고유 도메인·업무 맥락 반영(반도체 실무, 채용, 법률검토, 편집팀 등). 강사 예제와 과제 종류가 상이.
- 보일러플레이트 그대로 복붙한 성의 없는 제출 미발견.

### 신호 C (메타데이터) — 관찰됨, 부정 아님

| 사용자 | 관찰 | 판정 |
|--------|------|------|
| 양해찬 | 프롬프트 owner가 강사 계정(hiondal) | zip 제출을 강사가 압축해제·업로드 → **정상**(강사 확인 완료) |
| 류승재 | 파일 4개 modifiedTime 동일 | 일괄 업로드(드래그·동기화) → 정상 범위, 부정 근거 아님 |
| 이채림 | 파일 3개 modifiedTime 동일, 폴더/파일 owner 상이 | 일괄 업로드 + 개인계정 사용 → 부정 근거 아님 |

- 위 3건은 **단일 보조 신호**만 해당하며 내용 복제(신호 A) 정황이 없어 부정으로 판정하지 않음.
