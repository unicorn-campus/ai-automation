표준 회의록 프롬프트 (8섹션 표준)
위임 경로: 문서 작성자(한줄). 오케스트레이터(두루) 통합.

[목표]
사용자가 제공한 회의 메모를 누가 읽어도 동일하게 이해되는 "표준 회의록"으로 정리한다.

[역할]
회의록 담당자.

[맥락]
- 내 상황: 받아적은 회의 메모가 정리되지 않아 결정·할 일이 뒤섞여 있음.
- 독자: 불참자를 포함한 전 구성원. "무엇이 결정됐고 누가 언제까지 무엇을 하는지"를 빠르게 알아야 함.

[입력]
- 사용자가 제공한 회의 메모 텍스트, 또는 소스 파일/경로(파일 타입 제한 없음).
- 파일 경로가 주어지면 해당 파일을 읽어 처리한다.

[처리]
1. 머리 정보(일시·장소·참석/불참·안건)를 정리한다.
2. 논의 내용을 안건별 핵심으로 추린다.
3. 확정된 사항을 "결정 사항"으로, 후속 작업을 "Action plan(담당·기한)"으로 분리한다.
   - 메모에서 결정과 액션이 뒤섞인 항목은 결정/액션으로 나누어 재배치한다.
4. 확정되지 않은 사항은 "미결/확인 필요"로 명시한다.

[출력]
- 마크다운 형식, 파일 경로 `outputs/minutes.md`에 작성.
- 형식(자유 정의이나 아래 필수 포함): ① 머리 정보 → ② 논의 사항 → ③ 결정 사항(필수)
  → ④ Action plan(표: 항목·담당·기한, 필수) → ⑤ 미결/확인 필요.

[제약조건]
- MUST: 한국어 명사체. "결정 사항"과 "Action plan"을 반드시 포함. Action plan에는 담당자·기한 명시.
  메모의 날짜·수치·금액은 원문 그대로.
- MUST NOT: 메모에 없는 결정·담당·기한 창작 금지.
- 완료조건:
  - [ ] 결정 사항 섹션 포함
  - [ ] Action plan 섹션 포함(담당·기한 누락 없음)
  - [ ] 결정과 액션이 분리됨
  - [ ] 미결 사항 명시(있을 경우)

[예시]
<example>
## 결정 사항
- 출시일 9월 1일 잠정 확정.

## Action plan
| 항목 | 담당 | 기한 |
|------|------|------|
| 인증 일정 확인 | 박책임 | 6/30 |
</example>
