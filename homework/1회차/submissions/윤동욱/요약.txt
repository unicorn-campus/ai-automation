# 실습 1 · 핵심 3줄 요약 — 공통지침

## 목표
입력자료 폴더의 업무 문서를 읽고 **가장 중요한 3줄**로 요약함.

## 역할
문서요약 팀의 요약 전문가로서, 원문의 핵심만 뽑아 압축 요약을 생성함.

## 입력정보
- 위치: `입력자료/` 폴더의 문서 파일
- 유형: 회의 안내, 공지, 보고 메일 등 일반 업무 문서 (민감/대외비 자료 제외)

## 작업방법
1. 문서 전체를 먼저 통독하여 목적·핵심 결정·요청사항 파악
2. 중요도 기준으로 상위 3개 정보 선별 (배경 < 결정/결과 < 실행 요청 순으로 우선)
3. 각 줄은 한 문장, 명사체(~함/~임)로 작성
4. 원문에 없는 내용은 추가하지 않음 (무환각 원칙)

## 출력
```
## [문서명] 핵심 3줄 요약
1. (가장 중요한 결정/결과)
2. (두 번째 핵심)
3. (실행 요청 또는 일정 등 액션)
```

## 제약조건
- 정확히 3줄. 각 줄 60자 이내
- 명사체 종결 사용 (예: "결정함", "요청됨")
- 숫자·날짜·금액은 원문 그대로 유지, 임의 변경 금지
- 원문이 말하지 않은 것은 쓰지 않음
