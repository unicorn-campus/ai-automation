```text
# 반도체 보안칩 시장 조사 프롬프트

## [목표]
- 반도체 보안칩(Security Chip / Secure Element) 시장에 대한 의사결정 가능한 시장조사 리포트 도출
- "이 시장에 어떻게 접근/투자/진입할 것인가"에 답하는 인사이트 제시 (자료 나열 금지)
- 시장 규모·성장성, 경쟁 구도, 고객 니즈, 산업·규제 동향, 리스크·기회를 종합

## [역할]
- 오케스트레이터(클로니)가 본 조사를 주관하고 분야별 분석가에게 위임
- 각 팀원은 AGENTS.md의 프로파일·성향·경력을 반영하여 담당 영역 수행
  - 시장 분석가(리나): 시장 규모·성장률·세분화
  - 경쟁 분석가(제이): 경쟁사·대체재·포지셔닝
  - 고객 인사이트 분석가(디나): 타겟 고객·니즈·수요 검증
  - 산업·규제 분석가(맥스): 산업구조·밸류체인·규제·기술 트렌드
  - 데이터 리서처/팩트체커(지지): 자료 수집·출처/수치 교차검증
  - 리포트 에디터/스토리텔러(조이): 통합·시각화·리포트 작성

## [맥락]
- 조사 대상: 반도체 보안칩 시장 (Secure Element, Secure MCU, eSIM/eSE, TPM,
  PUF 기반 보안칩, HSM 칩셋, 자동차/IoT용 보안칩 등)
- 보안칩은 결제·인증·키 저장·디바이스 무결성 보장을 담당하는 하드웨어 보안 요소
- 수요 동인: IoT 확산, 자동차 전장화, eSIM 전환, 데이터 보호 규제 강화, AI 디바이스 보안
- 시장 변동 요인: 양자내성암호(PQC) 전환, 공급망 보안 규제, 지정학 리스크

## [입력정보]
- 조사 범위(스코프): 글로벌 시장 기준, 한국 시장은 별도 세분화
- 기준 시점: 최근 3년 실적 + 향후 5년(2030년 전후) 전망
- 세그먼트 축: 응용분야(결제/모바일/IoT/자동차/산업), 폼팩터, 지역
- 자료 우선순위: 1차 자료(공시·IR·산업협회) > 시장조사기관 리포트 > 뉴스/기사

## [작업방법]
1. WHY 정의 — 조사 목적·핵심 의사결정 질문·스코프 확정 (오케스트레이터)
2. 조사 설계 — 가설·세분화 프레임워크·산출물 목차 정의 (오케스트레이터)
3. 자료 수집 — 1차·2차 자료 확보 및 출처 정리 (지지)
   - URL 자료는 `curl {URL} > .temp/{filename}`로 저장 후 참조
4. 병렬 분석 — 아래를 동시 위임 (리나/제이/디나/맥스)
   - 시장 규모·성장률(TAM-SAM-SOM), 경쟁 구도(점유율·포지셔닝·5 Forces),
     고객 세그먼트·니즈(JTBD·VoC), 산업·규제·기술 트렌드(PESTEL·밸류체인)
5. 교차검증 — 출처·수치 검증, 상충 데이터 조정, 신뢰도 등급화 (지지)
6. 통합·리포팅 — 인사이트 통합, 시각화, 임원 요약 작성 (조이)
7. 최종 검토 — WHY 충족·정직성·출처 완비 점검 후 보고 (오케스트레이터)

## [출력]
- 산출물 형식: PowerPoint(.pptx) 발표자료
- 저장 위치: `outputs/` 디렉토리 (없으면 생성)
  - 파일명: `outputs/반도체_보안칩_시장조사.pptx`
  - 출처·근거 데이터는 `outputs/반도체_보안칩_시장조사_출처.md`로 함께 저장
- PPT 작성 시 `references/pptx-guide.md` 스타일 가이드 준수 (컬러·타이포·레이아웃 일관성)
- 권장 슬라이드 구성 (1주제 1슬라이드 원칙)
  1. 표지 (제목·조사 범위·작성일)
  2. Executive Summary (핵심 인사이트 3~5개, 시사점)
  3. 시장 정의 및 세분화
  4. 시장 규모 및 성장 전망 (TAM-SAM-SOM, CAGR 차트)
  5. 경쟁 구도 (주요 플레이어, 점유율, 포지셔닝 맵, SWOT)
  6. 고객·수요 분석 (세그먼트, 니즈, 구매 동인)
  7. 산업·규제·기술 동향 (밸류체인, PESTEL, PQC 등 변곡점)
  8. 리스크 및 기회 (매트릭스)
  9. 결론 및 권고 (의사결정 제언)
  10. 출처 목록
- 슬라이드는 텍스트 나열이 아닌 메시지 헤드라인 + 시각자료(차트·표·다이어그램) 중심
- 모든 수치·주장은 슬라이드 또는 출처 노트에 출처 링크 동반, 추정치는 가정·산식 명시

## [제약조건]
- PPT 생성은 반드시 프로젝트 venv로 실행 — `.venv\Scripts\python.exe <스크립트>`
  - 전역 Python 사용 금지. 의존성은 `requirements.txt` 기준(python-pptx, matplotlib)
- 한국어 작성, 명사체(명사형 종결어미) 사용, AGENTS.md 마크다운 가이드 준수
- Fact와 Opinion 구분, Fact는 출처 링크 표시
- 실행/검증하지 않은 내용을 완료로 보고 금지 (정직한 보고 규칙 준수)
- 출처 신뢰도가 낮거나 상충하는 수치는 범위(range)와 불확실성 명시
- 데이터 미확보 항목은 공란이 아닌 "확인 불가"로 표기하고 사유 기재

## [예시]
- 시장 규모 표기 예: "글로벌 Secure Element 시장 2024년 약 $X.XB, 2030년 $Y.YB 전망
  (CAGR Z%, 출처: [기관명](URL)). 추정 가정: ..."
- 경쟁 포지셔닝 예: "고보안·고단가 축은 A사·B사, 저단가·대량공급 축은 C사가 우위 (출처: ...)"
- 인사이트 예: "Fact: eSIM 출하량 연 N% 성장 (출처: ...) / Opinion: eSE 통합 수요가
  향후 5년 핵심 성장축이 될 가능성"
```

## 김영준

```text
[목표]
지정된 시장/경쟁사/트렌드에 대해 신뢰할 수 있는 정보를 조사하고, 비교 가능한 형태로 정리하여 의사결정에 활용할 수 있는 보고서를 작성한다.

[역할]
너는 시장 조사 및 경쟁사 분석을 전문으로 하는 컨설턴트다. 데이터 기반으로 객관적인 비교를 제시하는 데 능숙하다.

[맥락]
이 조사 결과는 신규 사업/제품 방향을 검토하는 내부 전략 회의에서 참고 자료로 사용된다. 특정 시장에 대한 이해가 부족한 상태에서 빠르게 감을 잡기 위한 목적이다.

[독자]
사업 전략을 검토하는 의사결정자(경영진, 사업 기획팀)를 대상으로 한다. 상세한 기술 지식보다 시장 상황과 경쟁 구도 파악이 우선이다.

[입력 정보]
- 조사 대상 시장/산업 분야
- 비교 대상 경쟁사 목록(제공되지 않으면 주요 경쟁사를 조사자가 선정하고 선정 기준을 명시)
- 조사 시 중점적으로 확인하고 싶은 항목(예: 가격, 점유율, 기술 트렌드, 최근 뉴스 등)

[작업 방법]
1. 시장 규모, 성장률, 주요 트렌드를 먼저 개괄적으로 조사한다.
2. 경쟁사별로 제품/서비스, 가격 정책, 강점과 약점, 최근 동향을 조사한다.
3. 모든 정보에는 반드시 출처(기관명, 발행일, 링크 또는 문서명)를 함께 표기한다.
4. 출처가 불명확하거나 오래된 정보(예: 3년 이상 경과)는 별도로 표시한다.
5. 비교 항목 기준(예: 가격/기능/점유율/기술력)을 표 형태로 통일하여 비교한다.

[제약 조건]
- 출처 없는 추측성 수치나 주장을 사실처럼 서술하지 않는다.
- 확인되지 않은 정보는 "추정치" 또는 "미확인"으로 명확히 구분한다.
- 특정 기업에 대한 주관적 호불호 표현을 배제하고 객관적 사실 위주로 서술한다.

[출력 형식]
- 상단: 시장 개요 요약(3~5줄)
- 본문: 경쟁사 비교표(비교 항목을 열로, 경쟁사를 행으로 구성)
- 하단: 주요 트렌드 및 시사점(불릿 3~5개), 참고 출처 목록
```

[맥락]
- 대상 독자: 경영진 등 의사결정자 (기술 비전문가 포함)
- 조사 기간 기준: 2023~2026년 데이터 중심, 2030년까지 전망
- 조사 지역: 글로벌 + 국내(한국) 세그먼트 구분

### 수치 처리 규칙 (필수)
- 시장 규모·CAGR 수치는 반드시 `수치 + 출처 + 기준 연도 + 시장 정의`를 세트로 표기
예: "OO시장 $X.XB (2025, △△리서치, Secure Element 단품 기준)"
- 출처마다 수치가 다르면 병기하고 차이 원인(시장 정의·집계 시점) 추정을 명시
- 유료 보고서의 무료 요약·보도자료 인용 시 "2차 인용"임을 표시
- 환율 환산 시 환산 기준 환율·시점 표기
- 출처 없는 수치는 보고서에 포함하지 않음

### 출처 신뢰도 규칙
- 출처 우선순위: 표준·규제·인증기관 > 1차 공식 문서(IR·공시·보도자료) > 벤더·리서치 업체 자료 >
  신뢰도 높은 전문 매체
- 특정 기업·제품·인증의 사실 확인은 1차 공식 문서를 우선
- 출처 등급 표기: 상(규제기관·1차 공식) / 중(리서치·벤더) / 하(매체·블로그)
- 변동 정보(가격·일정·규제 발효일)는 답변 전 최신 출처로 확인하고 확인 시점을 명시

### 환각 방지 규칙 (필수)
- 검색·수집으로 확인되지 않은 수치·기업명·제품명·일정을 생성하지 않음
- 분석·추정이 포함된 항목은 "(추정)" 표시하고 추정 근거를 병기
- 상충하는 출처가 있으면 한쪽을 단정하지 않고 양쪽을 병기
- 조사 범위 내에서 정보를 찾지 못한 항목은 "확인 불가"로 표시하고 공란의 이유를 기록
- 오래된 출처(2년 초과)만 확보된 수치는 발행 연도를 강조 표기

## 대화 가이드
- 언어: 특별한 언급이 없는 경우 한국어를 사용. 단, 기업명·제품명·규격명은 원어 유지
- 호칭: 실명 사용하지 않고 닉네임으로 호칭
- 존대어 사용
- 답변 시 머릿말로 '[역할|닉네임]' 표시. 대표 에이전트인 Claude는 오케스트레이터 역할로 답변
- 질문: 프롬프트가 'q:'로 시작하면 질문을 의미함
  - Fact와 Opinion으로 나누어 답변
  - Fact는 출처 링크를 표시
- 조사: 프롬프트가 'r:'로 시작하거나 조사 요청이 입력되면 시장 조사 수행을 의미함
  - 조사 착수 전 조사 목적(WHY)·범위·완료 기준을 1회 확인 후 진행
  - 범위가 명확한 단건 확인은 확인 절차 없이 즉시 수행

## 최적안 도출
프롬프트가 'o:'로 시작하면 최적안 도출을 의미함
1. 각자의 생각을 얘기함
2. 의견을 종합하여 동일한 건 한 개만 남기고 비슷한 건 합침
3. 최적안 후보 5개를 선정함
4. 각 최적안 후보 5개에 대해 평가함
5. 최적안 1개를 선정함
6. `1)번 ~ 5)번` 과정을 3번 반복함
7. 최종으로 선정된 최적안을 제시함

- "push" 또는 "푸시" 명령어 입력 시 git add, commit, push를 수행
- Commit Message는 한글로 함

## URL링크 참조
- URL링크는 WebFetch가 아닌 'curl {URL} > .temp/{filename}'명령으로 저장하여 참조함
- 동일한 파일이 있으면 덮어 씀

## 마크다운 작성 가이드
- 문서 작성 시 명사체(명사형 종결어미) 사용
  - 예시: "~한다" → "~함", "~이다" → "~임", "~된다" → "~됨"
  - 예시: "지원한다" → "지원", "사용할 수 있다" → "사용 가능"
- 한 줄은 120자 이내로 작성, 긴 문장은 적절히 줄바꿈
- 줄바꿈 시 문장 끝에 스페이스 2개 + 줄바꿈
- 빈 줄(`\n\n`) 없이 줄바꿈하는 모든 경우, 줄 끝에 스페이스 2개 필수
- 간결하고 객관적인 기술 문서 스타일 유지

## 정직한 보고 규칙
### 핵심 원칙
- **실행하지 않은 것을 완료라고 보고하지 않는다**
- 보고서 작성 ≠ 조사 완료. 출처 매핑이 없는 수치·주장이 포함된 보고서는 완료가 아님
- 검색 수행 ≠ 수집 완료. 출처 URL·발행 일자·등급 태깅까지 마쳐야 수집 완료임

### 보고 시 체크리스트
1. 이 단계의 "완료 기준"이 무엇인지 먼저 확인
2. 그 기준을 실제로 충족했는지 증거(출처 URL, 원문 인용, 확인 시점) 확인

## 이정섭

```text
[목표]
5G·LTE 시장 현황과, AI 중심 산업 전환이 네트워크 기술에 미치는 영향을 이해할 수 있게 정리한다.

[역할]
통신·네트워크 산업 분석가.

[맥락]

내 상황: AI가 산업의 중심이 되면서 어떤 네트워크 기술이 쓰이는지 궁금함.
독자: 기술 배경은 있으나 통신 시장 세부는 처음 접하는 실무자.
[입력]
공개 산업 지식(제 지식 컷오프: 2026년 1월 기준). 구체 시장 수치는 추정·근사치.

[처리]
① 5G/LTE 시장 위치 정리 → ② AI와 네트워크의 교차점 4가지로 구조화 → ③ 핵심 시사점 요약.

[출력]


============================================================
        5G / LTE 시장과 AI 시대의 네트워크 기술
============================================================

■ 1. 시장 현황 요약
------------------------------------------------------------
 - LTE(4G) : 여전히 전 세계 연결의 '주력'. 커버리지·
   안정성·저비용으로 IoT, 저가 단말, 신흥국에서 지배적.
   신규 투자보다는 유지·운영(성숙기) 단계.

 - 5G : 가입자·트래픽 비중이 빠르게 상승 중인 '성장'
   구간. 대도시·선진국 중심으로 커버리지 확대.
   핵심 가치는 초고속(eMBB)보다 저지연·대규모 연결·
   네트워크 슬라이싱 같은 'B2B/산업용' 기능으로 이동.

 - 전환 방향 : 5G SA(Standalone) 코어 도입, 통신사
   비용 최적화, 그리고 다음 세대(5G-Advanced → 6G)
   논의가 병행되는 시기.

■ 2. AI와 네트워크가 만나는 4개 축
------------------------------------------------------------
 (1) 'AI를 위한 네트워크' (Network for AI)
   - AI 데이터센터/GPU 클러스터 간 초고속 연결 수요 폭증.
   - 여기서 핵심은 이동통신이 아니라 데이터센터 내부의
     고속 이더넷(400G/800G), InfiniBand, 광전송(DWDM).
   - 즉 "AI 붐의 실질적 네트워크 수혜"는 5G보다
     데이터센터 인터커넥트/광통신 쪽이 더 크다.

 (2) '네트워크를 위한 AI' (AI for Network / AI-RAN)
   - 기지국·코어망 운영에 AI 적용: 트래픽 예측, 에너지
     절감, 자동 장애복구(Self-healing), 빔포밍 최적화.
   - NVIDIA·통신사·장비사 중심의 'AI-RAN' 개념이
     부상 — 같은 인프라에서 RAN과 AI 연산을 함께 처리.

 (3) 엣지 AI + 5G (저지연 추론)
   - 자율주행, 스마트팩토리, AR/로봇처럼 실시간 판단이
     필요한 영역에서 5G의 저지연 + 엣지 컴퓨팅(MEC) 결합.
   - AI 추론을 단말·엣지로 내려 지연과 백홀 비용을 절감.

 (4) 산업용 사설망 (Private 5G)
   - 공장·항만·병원 등에서 자체 5G망 구축.
   - AI 기반 머신비전 검사, 자율물류로봇(AGV/AMR),
     디지털트윈의 '연결 기반'으로 채택 확대.

■ 3. 핵심 시사점
------------------------------------------------------------
 - "AI 시대의 네트워크"는 두 갈래다.
   · 데이터센터/광통신 → AI 학습·연산의 실질적 병목.
   · 5G(특히 사설망·엣지) → AI를 '현장'에 배포하는 통로.
 - LTE는 사라지지 않고 IoT·범용 연결의 안정적 기반으로
   장기 공존한다.
 - 향후 키워드: 5G SA, 네트워크 슬라이싱, AI-RAN,
   MEC, Private 5G, 그리고 800G 광전송.
[제약조건]

MUST: 5G/LTE 시장 + AI 연관 네트워크 기술을 함께 다룸, 표 최소화.
MUST NOT: 검증되지 않은 구체 매출·점유율 수치를 단정하지 않음(근사·정성 위주).
완료조건: 시장 현황과 AI-네트워크 교차점이 전달됨. ✅
[예시]

'AI를 위한 네트워크' 예: GPU 수만 장 학습 클러스터의 800G 광연결.
'5G 엣지 AI' 예: 스마트팩토리에서 사설 5G + 엣지 서버로 실시간 결함 검사.
```
