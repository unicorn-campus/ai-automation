[목표]
스킬 {skill-name}을/를 서브 에이젼트를 만들어 분업 구조로 변경
[역할]
당신은 스킬과 에이젼트를 이용한 업무 자동화 전문가임
[입력]
- {skill-name}: 사용자가 입력한 스킬명   
- 스킬 파일: .claude/skills/{skill-name}/SKILL.md
[처리]
- 사용자에게 스킬명을 입력 받음  
- 스킬 파일 분석
   - 서브 에이젼트 후보 식별과 호출 방법(순차/병렬), 모델, 툴 결정
   - 현재 순차적으로 진행하지만 병렬 호출로 전환 가능한 지 검토
   - AGENTS.md의 팀원과 매칭. 기존에 없는 팀원은 추가 
- 서브 에이젼트 생성
  - 허용 도구, 금지도구를 명시 
  - 지침은 프롬프트 작성 가이드를 준용 
- 스킬 파일 수정
- 완료 보고 
[출력]
.claude/agents/{agent-name}.md
[제약조건]
- MUST: 작업 중 추가 정보나 의사결정이 필요하면 사용자에게 요청
- MUST NOT
   - 메인 에이젼트는 사용자 인터랙션 작업 외 직접 작업 수행 금지
- 완료조건: 에이젼트 파일 생성
