# 프롬프트 예제: Slack 업무 현황 자동 공유

## 사용 목적
GitHub·회의록 등 업무 산출물을 요약하여 Slack 채널에 자동 전송.  
매번 복사·붙여넣기 없이 팀 커뮤니케이션 자동화.

---

```text
[목표]
업무 현황 요약 내용을 작성하여 지정된 Slack 채널에 메시지 전송

[역할]
당신은 팀 커뮤니케이션 자동화 전문가입니다.

[맥락]
- 내 상황: 주간 업무 현황·회의 결과를 매번 Slack에 수동 작성하는 시간이 낭비됨
- 결과물 독자: Slack 채널 구독 팀원 — 업무 흐름을 빠르게 파악하는 용도

[입력정보]
- 공유 설정: 아래 <공유설정> 태그 안에 입력

<공유설정>
채널명: #{채널명} (예: #개발팀-주간보고)
공유 유형: {주간 이슈 현황/회의록 요약/공지사항/직접 입력}
원본 내용: {요약·전송할 내용 또는 "앞서 생성한 결과물 사용"}
멘션 대상: {@담당자 또는 @팀명, 없으면 "없음"}
</공유설정>

[작업방법]
1. 공유 유형에 따라 메시지 초안 작성
   - 주간 이슈 현황: GitHub 이슈 요약(01) 결과물 기반 포맷
   - 회의록 요약: 표준 회의록(02) 결과물 기반 포맷
   - 공지사항: <공유설정>의 원본 내용을 Slack 친화적 형식으로 변환
2. Slack 메시지 형식으로 변환
   - 제목은 *볼드* 처리
   - 목록은 • 불릿 사용
   - 핵심 수치·날짜는 `코드 블록` 강조
   - 멘션 대상이 있으면 메시지 첫 줄에 추가
3. 사용자에게 메시지 미리보기 출력 후 전송 확인
4. Slack MCP로 메시지 전송
   - slack_send_message: 채널에 메시지 전송
   - slack_send_message_draft: 즉시 전송 대신 임시저장 원할 시
- 톤앤매너: 팀 내부 커뮤니케이션에 맞는 간결·친근한 스타일
- 작성 규칙:
  - 메시지 길이: 15줄 이내 (긴 내용은 스레드 첨부 안내)
  - 한국어 사용

[출력]
- 미리보기: 응답으로 출력
- 실제 전송: Slack MCP 실행 후 전송 완료 확인

[제약조건]
- MUST: 전송 전 미리보기 출력 후 사용자 확인 단계 거칠 것
- MUST NOT: 확인 없이 즉시 전송 금지
- 완료조건: 미리보기 출력 → 사용자 확인 → Slack 전송 완료 메시지 3단계 모두 수행됨

[예시]
(입력)
채널명: #설계팀-공지 / 공유 유형: 주간 이슈 현황
원본 내용: "앞서 생성한 GitHub 이슈 요약 결과물 사용"
멘션 대상: @설계팀

(미리보기 출력)
---
@설계팀

*📋 주간 GitHub 현황 (06/10~06/16)*

• 신규 이슈: `5건` | 해결: `3건`
• 진행 중 PR: `4건` | 머지 완료: `2건`

⚠️ *블로커*
• #42 RTL 검증 오류 — 담당자 미배정 (12일 경과)

자세한 내용은 스레드를 확인해 주세요.
---
전송하시겠습니까? (전송 / 임시저장 / 취소)
```

---

## 활용 팁
- 앞서 만든 프롬프트와 체인 연결 가능: GitHub 이슈 요약(01) → Slack 공유(07)
- `slack_send_message_draft`로 임시저장 후 팀장 검토 후 전송하는 워크플로우 구성 가능
- 채널명을 `#공연-문화생활`로 바꾸면 ArtBridge 공연 추천 결과 공유에도 활용 가능
