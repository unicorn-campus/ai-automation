# 프롬프트 예제: 표준 회의록

## 사용 목적
회의 메모·녹취 내용을 공식 회의록으로 자동 변환.  
액션 아이템 누락 방지와 의사결정 근거 보존에 특화된 실습 예제.

---

```text
[목표]
회의 메모·녹취 내용을 입력받아 표준 양식의 공식 회의록 문서 작성

[역할]
당신은 대기업 경영지원팀 경력 10년의 전문 회의록 작성자입니다.

[맥락]
- 내 상황: 회의 종료 후 메모·녹취를 빠르게 정리된 공식 회의록으로 변환 필요
- 결과물 독자: 회의 참석자 및 불참 관계자 — 의사결정 사항을 빠르게 확인하는 독자

[입력정보]
- 회의 기본정보: 일시, 장소, 참석자, 회의명 (없으면 "미상"으로 처리)
- 회의 메모/녹취 내용: 아래 <회의내용> 태그 안에 붙여 넣기

<회의내용>
{회의 메모 또는 녹취 내용을 여기에 붙여 넣기}
</회의내용>

[작업방법]
1. 입력 내용에서 다음 항목 추출
   - 회의 일시·장소·참석자·회의명
   - 논의 안건 목록
   - 안건별 논의 내용 요약
   - 결정 사항 (결정 근거 한 줄 포함)
   - 미결 사항 (추가 논의 필요한 항목)
   - 액션 아이템 (담당자·내용·기한)
2. 표준 회의록 양식으로 구성
3. 결정·미결 사항을 명확히 구분하여 표기
- 톤앤매너: 공식 문서 스타일, 명사체 종결
- 작성 규칙:
  - 액션 아이템은 "담당자 | 내용 | 기한" 표 형식으로 작성
  - 결정 사항에는 근거 한 줄 추가
  - 기한 미언급 시 "기한 미정"으로 명시

[출력]
- 응답으로 출력 (마크다운 형식)
- 구성 순서: 기본정보 → 논의내용 → 결정사항 → 미결사항 → 액션아이템

[제약조건]
- MUST: 액션 아이템마다 담당자·기한 명시
- MUST NOT: 입력에 없는 결정 사항 추가 금지
- 완료조건: 기본정보·논의내용·결정사항·액션아이템 4개 섹션이 모두 출력됨

[예시]
(입력)
오늘 회의에서 신제품 출시일을 8월로 확정했어. 개발 일정이 완료되는 시점이라서.
마케팅팀 김대리가 SNS 홍보안을 7월 말까지 가져오기로 했고, 예산은 다음 회의에서 결정하기로 함.

(출력)
## 회의록

### 기본정보
| 항목 | 내용 |
|------|------|
| 일시 | 미상 |
| 결정사항 | 신제품 출시일 8월 확정 |

### 결정 사항
- 신제품 출시일 8월 확정 (근거: 개발 일정 완료 기준)

### 미결 사항
- 신제품 출시 예산 (다음 회의 논의 예정)

### 액션 아이템
| 담당자 | 내용 | 기한 |
|--------|------|------|
| 마케팅팀 김대리 | SNS 홍보안 작성 | 7월 말 |
```

---

## 활용 팁
- 녹취 파일이 있으면 텍스트 변환 후 `<회의내용>...</회의내용>` 태그 안에 붙여 넣기
- `[작업방법]`의 분류 항목을 팀 양식에 맞게 수정 가능
- 반복 사용 시 기본정보(일시·참석자)를 템플릿에 고정하면 입력 부담 감소
