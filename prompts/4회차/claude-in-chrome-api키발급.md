# 프롬프트 예제: Claude in Chrome으로 API 키 자동 발급 (비개발자용)

## 사용 목적
서비스 이름 하나만 입력하면, Claude가 크롬 브라우저를 대신 조작하여 그 서비스의 **API 키**(=서비스를  
다른 프로그램에서 쓸 수 있게 해주는 비밀번호 같은 출입증)를 발급받아 안전한 곳에 저장해 주는 프롬프트.  
개발자 페이지가 영어·전문용어로 복잡해 어디서 키를 만드는지 모르는 비개발자를 위한 자동화 예제.

---

```text
[목표]
사용자가 입력한 서비스의 API 키를 Claude in Chrome(브라우저 자동조작)으로 웹사이트에서 발급받아
`~/.claude/secrets/.env` 파일에 안전하게 저장

[역할]
당신은 수백 개 온라인 서비스의 개발자 페이지 구조를 꿰고 있으며,
비개발자를 대신해 브라우저를 안전하게 조작해 API 키 발급을 대행하는 웹 자동화 도우미입니다.

[맥락]
- 내 상황: 나는 비개발자임. OpenAI·카카오 같은 서비스의 API 키가 필요한데,
  개발자 페이지가 영어와 낯선 용어로 가득해 어디서 어떻게 키를 만드는지 막막함
- 결과물 독자: 나 자신 — 발급받은 키를 자동화 도구(예: MCP 서버)에 그대로 붙여 바로 사용하려 함

[입력정보]
- 발행 대상 서비스: 사용자에게 질문하여 수집 (예: OpenAI, Anthropic, 구글, 카카오, 네이버, 공공데이터포털)
- 사용 브라우저: Claude in Chrome MCP (사용자가 평소 쓰는, 이미 로그인된 크롬을 그대로 사용)
- 키 저장 위치: `~/.claude/secrets/.env`
- 주요 서비스 발급 페이지 (목록에 없으면 "{서비스명} API 키 발급 공식 페이지"로 웹 검색하여 결정)

  | 서비스 | 발급 페이지 주소 | 저장 변수명(예) | 사전 준비 |
  |--------|----------------|----------------|----------|
  | OpenAI | https://platform.openai.com/api-keys | `OPENAI_API_KEY` | 없음 |
  | Anthropic(Claude) | https://console.anthropic.com/settings/keys | `ANTHROPIC_API_KEY` | 없음 |
  | 구글 AI Studio(Gemini) | https://aistudio.google.com/apikey | `GOOGLE_API_KEY` | 없음 |
  | 카카오 개발자 | https://developers.kakao.com/console/app | `KAKAO_REST_API_KEY` | 앱 먼저 생성 |
  | 네이버 개발자센터 | https://developers.naver.com/apps | `NAVER_CLIENT_ID` 등 | 앱 먼저 생성 |
  | 공공데이터포털 | https://www.data.go.kr | `DATA_GO_KR_KEY` | 활용신청 필요 |

[작업방법]
1. 발급받을 서비스 질문
   - "어떤 서비스의 API 키를 발급할까요?"라고 묻고 서비스명 1개를 받음

2. 발급 페이지 결정
   - 위 표에 있으면 해당 주소 사용
   - 표에 없으면 웹 검색으로 그 서비스의 공식 API 키 발급 페이지 주소를 찾음

3. Claude in Chrome으로 페이지 열기 (해당 발급 페이지 주소로 이동)

4. 로그인 상태 확인
   - 이미 로그인되어 있으면 다음 단계로 진행
   - 로그인이 안 되어 있으면: "브라우저 화면에서 직접 로그인해 주세요. 끝나면 알려 주세요"라고 안내 후 대기
   - 로그인 도중 2단계 인증·문자인증·캡차(로봇 확인)가 나오면: 대신 처리하지 않고 사용자에게 직접 진행 요청 후 대기

5. (필요 시) 앱·프로젝트 먼저 만들기
   - 카카오·네이버처럼 키 발급 전 "앱/프로젝트" 생성이 필요한 서비스면:
     - 기존에 만든 것이 있으면 그대로 사용
     - 없으면 새로 생성 (이름은 `auto-ai-{오늘날짜}` 형식, 예: `auto-ai-20260618`)

6. API 키 만드는 메뉴로 이동
   - 화면에서 "API Keys", "Create new key", "키 발급", "인증키" 같은 버튼·메뉴를 찾아 이동

7. 새 키 생성
   - 화면에 이미 만들어 둔 키가 보이고 그 값을 알고 있으면, 새로 만들지 사용자에게 확인 (불필요한 키 양산 방지)
   - "새 키 만들기" 버튼을 눌러 키를 생성
   - 키 이름(라벨)을 입력하는 칸이 있으면 `auto-ai-{오늘날짜}` 입력

8. 생성된 키 값 안전하게 가져오기 (★ 가장 중요)
   - 많은 서비스는 키를 화면에 **딱 한 번만** 보여주므로, 생성 직후 즉시 처리
   - 화면 전체 캡처나 페이지 전체 글자 읽기로 가져오지 말 것
   - 키가 적힌 칸 하나의 값만 정확히 추출(예: 페이지의 '복사' 버튼 이용 또는 해당 칸 값만 읽기)하여
     곧바로 9번에서 파일에 저장
   - 키 값으로 보이는 칸이 여러 개이거나 어느 것이 키인지 확실하지 않으면, 추측해 저장하지 말고 사용자에게 확인
   - 추출한 키 전체 값을 대화창이나 보고에 다시 그대로 출력하지 말 것

9. 키 저장
   - 저장 폴더 `~/.claude/secrets/` 가 없으면 먼저 생성
   - `~/.claude/secrets/.env` 파일에 `{변수명}={키값}` 형식으로 추가 (변수명은 위 표 참고, 없으면 대문자로 정함)
   - 같은 변수가 이미 있으면 "기존 키를 새 키로 덮어쓸까요?"라고 물어 확인 후 처리
   - 키가 실수로 깃(소스 저장소)에 올라가지 않도록, 저장 폴더가 깃 프로젝트 안이면 `.gitignore`에 등록

10. 발급 직후 키 유효성 확인 (가능한 경우)
   - 읽기 전용 요청 1건으로 키가 실제로 동작하는지 확인 (예: OpenAI는 모델 목록 조회)
   - 동작 확인이 어려운 서비스면, 보고에 "발급·저장 완료, 유효성 미검증"이라고 솔직히 명시

11. 결과 보고
   - 어떤 서비스의 키를 발급했는지, 저장 위치, 저장 변수명, 키 미리보기(앞 4자리 + `****`),
     유효성 확인 결과, 다음에 할 일을 한국어로 요약

- 톤앤매너: 비개발자 눈높이로 친절하게. 전문용어는 풀어서 한 줄 설명을 덧붙임
- 작성 규칙:
  - 각 단계 시작 시 "지금 ○○ 페이지를 여는 중입니다"처럼 무엇을 하는지 한국어로 알려줌
  - 막혀서 사용자 도움이 필요하면 추측하지 말고 멈춰서 물어봄

[출력]
- 발급된 API 키가 `~/.claude/secrets/.env`에 저장됨
- 응답으로 발급 결과 요약 출력 (서비스명 / 저장 위치 / 변수명 / 마스킹된 키 미리보기 / 유효성 / 다음 단계)

[제약조건]
- MUST:
  - Claude in Chrome MCP로 실제 발급 페이지를 조작하여 키를 발급
  - 키는 생성 직후 즉시 `~/.claude/secrets/.env`에 저장 (한 번만 표시되는 서비스 대비)
  - 키 값은 칸 하나만 정확히 읽고, 보고 시에는 앞 4자리 + `****`로 마스킹
  - 저장 변수가 기존에 있으면 덮어쓰기 전 사용자에게 확인
- MUST NOT:
  - 사용자 비밀번호를 대신 입력하지 않음 (로그인은 반드시 사용자가 직접)
  - 2단계 인증·문자인증·캡차를 대신 처리하지 않음
  - 유료 플랜 결제·결제수단 등록 등 돈이 드는 행위를 대신하지 않음
  - 키 발급과 무관한 계정 설정 변경 금지
  - 키 발급 페이지 전체를 화면 캡처하거나 전체 텍스트로 읽어 키를 노출하지 않음
  - 추출한 키 전체 값을 대화·보고·로그에 그대로 다시 출력하지 않음
- 완료조건:
  - `~/.claude/secrets/.env` 파일에 해당 서비스 키 변수가 실제로 저장됨 (파일에서 변수 존재 확인)
  - 유효성 확인을 했으면 그 결과를, 못 했으면 "미검증"임을 보고에 명시 (확인 안 한 것을 된 것처럼 보고 금지)
  - 발급 결과(서비스·변수명·저장위치·마스킹 키)를 응답으로 보고

[예시]
(입력)
발행 대상 서비스: OpenAI

(진행 안내 예시)
1) 지금 OpenAI 키 발급 페이지(platform.openai.com/api-keys)를 여는 중입니다
2) 로그인 상태를 확인했습니다 — 이미 로그인되어 있어 그대로 진행합니다
3) 'Create new secret key'를 눌러 `auto-ai-20260618` 이름으로 새 키를 만들었습니다
4) 키를 `~/.claude/secrets/.env`의 `OPENAI_API_KEY`에 저장했습니다
5) 모델 목록 조회로 키가 정상 동작함을 확인했습니다

(결과 보고 예시)
✅ OpenAI API 키 발급 완료
- 저장 위치: `~/.claude/secrets/.env`
- 변수명: `OPENAI_API_KEY`
- 키 미리보기: `sk-p****` (전체 값은 보안상 표시하지 않음)
- 유효성: 확인됨 (모델 목록 조회 성공)
- 다음 단계: 이제 이 키를 쓰는 자동화(예: gpt-image-mcp.md)를 바로 실행할 수 있습니다
- 보안 주의: 이 대화 세션을 타인과 공유하지 마세요. 키 유출이 의심되면 즉시 해당 서비스에서 키를 폐기(재발급)하세요
```

## 활용 팁
- 평소 쓰는 크롬에 **미리 로그인**해 두고 실행하면 로그인 단계가 생략되어 가장 매끄러움
- 발급된 키는 `~/.claude/secrets/.env`에 저장되므로, 같은 폴더를 읽는 다른 프롬프트  
  (예: `gpt-image-mcp.md`)가 키를 바로 가져다 사용 가능 — 발급→활용 연계
- 표에 없는 서비스도 "○○ API 키 발급해 줘"처럼 입력하면 공식 페이지를 검색해 진행
- OpenAI처럼 키를 **한 번만** 보여주는 서비스도, 이 프롬프트가 생성 즉시 저장하므로 놓칠 걱정 없음
- 2단계 인증·결제 단계는 보안상 대신 처리하지 않으므로, 그 부분만 사용자가 직접 진행하면 됨
- 보안 원칙: 키 전체 값은 화면·보고에 노출하지 않고 마스킹 처리하며, 유출 의심 시 즉시 재발급 권장
- 저장 폴더(`~/.claude/secrets/`)가 클라우드 동기화(OneDrive 등)·자동 백업 대상이 아닌지 확인 권장  
  (동기화 폴더에 두면 키가 외부로 함께 올라갈 수 있음)
