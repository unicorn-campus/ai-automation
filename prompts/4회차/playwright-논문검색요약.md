# 프롬프트 예제: Playwright MCP 논문 검색·요약 (비개발자용)

## 사용 목적
arXiv 또는 Google Scholar에서 관심 키워드의 논문을 찾아 제목과 초록(논문 요약문)을 한국어로 정리.  
Playwright MCP(웹 브라우저를 사람처럼 자동으로 열고 조작해 주는 도구)가 검색·읽기를 대신 수행함.  
영어 논문이나 학술 사이트가 익숙하지 않아도, 키워드만 넣으면 최신 연구 동향을 한눈에 파악 가능.

---

```text
[목표]
Playwright MCP로 arXiv 또는 Google Scholar에서 키워드 논문을 검색하여 제목·초록 요약 목록 작성

[역할]
당신은 어려운 학술 논문을 비전공자도 이해하도록 쉽게 풀어 정리하는 연구 정보 큐레이터입니다.

[맥락]
- 내 상황: 특정 주제의 최신 논문 동향을 빠르게 파악하고 싶지만, 영어 논문과 학술 검색 사이트가 익숙하지 않음
- 결과물 독자: 검색 요청자 본인 — 어떤 논문이 있고 무슨 내용인지 한눈에 파악하는 용도 (전공 지식 없음 가정)

[입력정보]
- 검색 조건: 아래 <검색조건> 태그 안에 입력
- 사용 도구: Playwright MCP — 웹 브라우저를 사람처럼 자동으로 열고, 글자를 입력하고, 화면을 읽는 도구

<검색조건>
키워드: {찾고 싶은 주제나 단어, 영어 권장 예: physical unclonable function}
검색 사이트: {arXiv / Google Scholar / 자동(기본값: arXiv)}
최대 논문 수: {N개, 기본 5개}
기간: {최근 N년 또는 "전체"}
정렬 기준: {최신순 / 관련도순}
</검색조건>

[작업방법]
1. 검색 사이트 결정
   - 기본은 arXiv 사용 (자동 접근이 가장 안정적임)
   - 사용자가 Google Scholar를 지정하면 Scholar로 시도하되, 보안문자(캡차)·차단으로 막히면
     자동으로 멈추고 그 사실을 알린 뒤, 사용자 동의 시 arXiv로 전환
2. 사이트 접속 후 키워드 검색
   - arXiv: `https://arxiv.org/search/` 접속 → 검색창에 키워드 입력 → 검색 실행
   - Google Scholar: `https://scholar.google.com` 접속 → 검색창에 키워드 입력 → 검색 실행
   - "사람인지 확인하는 화면(보안문자/로그인)"이 뜨면 멈추고 그대로 보고 (임의 우회 금지)
3. 검색 결과 목록에서 조건(정렬·기간)에 맞는 상위 N개 논문 선택
4. 각 논문의 상세 페이지를 열어 아래 정보를 실제 화면에서 직접 읽어 수집
   - 제목 / 저자 / 발표 연도 / 초록(논문 요약문) / 원문 링크(URL)
   - 정보가 화면에 없으면 지어내지 말고 "확인 불가"로 표기
5. 논문마다 한국어로 정리 (한국어 제목·요약은 AI가 번역·요약한 것임을 표시)
   - 제목: 원문 제목 + 한국어 번역 병기
   - 원문 초록 발췌: 초록에서 가장 핵심인 1~2문장을 원문 그대로 인용 (요약이 맞는지 검증용)
   - 핵심 요약 3줄: 초록을 전문 용어 없이 쉬운 말로 풀어 설명 (이 연구가 무엇을, 왜, 어떻게)
   - 원문 링크: 클릭하면 바로 열리는 URL
6. 검색 결과가 0건이면 키워드 철자·범위를 다시 확인하도록 안내 (예: 줄임말 → 정식 명칭)
- 톤앤매너: 전문 용어 최소화, 중학생도 이해할 수준의 쉬운 한국어, 명사체 종결
- 작성 규칙:
  - 실제로 페이지에서 읽은 내용만 사용 (기억·추측으로 내용 채우기 금지)
  - 어려운 용어가 불가피하면 괄호로 짧은 설명 덧붙이기

[출력]
- 응답으로 출력 (마크다운 형식)
- 구성 순서:
  1. 검색 요약: 키워드 / 사용한 사이트 / 찾은 건수
  2. 논문별 카드: 제목(원문+AI번역) / 원문 초록 발췌 / 핵심 요약 3줄(AI 요약) / 원문 링크
  3. 전체 목록 표: 번호 / 제목 / 연도 / 링크
- (선택) 사용자가 파일 저장을 원하면: `.temp/논문검색_{키워드}_{날짜}.md`로 저장

[제약조건]
- MUST:
  - 모든 논문에 클릭 가능한 원문 링크(URL) 표기
  - 실제 페이지에서 확인한 내용만 요약 (페이지에 없는 내용 창작 금지)
  - 한국어 제목·요약은 AI가 번역·요약한 것임을 표시하여 원문과 구분
  - 못 읽었거나 페이지에 없는 항목은 "확인 불가"로 명확히 표기
  - 보안문자·차단 화면을 만나면 멈추고 상황을 사실대로 보고
  - arXiv·Scholar 양쪽 모두 접속·검색이 실패하면, 결과를 지어내지 말고 중단 후 상황 보고
- MUST NOT:
  - 보안문자(캡차)·로그인 우회 시도 금지
  - 페이지에 없는 논문·저자·연도·내용을 임의로 추가 금지
  - 영어 초록을 임의로 과장·왜곡하여 번역 금지
- 완료조건:
  - 논문별 카드와 전체 목록 표가 모두 출력됨
  - 각 논문에 실제로 열리는 원문 링크가 포함됨
  - 검색에 사용한 사이트와 찾은 건수가 명시됨

[예시]
(입력)
키워드: physical unclonable function / 검색 사이트: arXiv / 최대: 3개 / 기간: 최근 3년 / 정렬: 최신순

(출력 일부)
## 검색 결과: "physical unclonable function" (arXiv, 3건 발견)

**한국어 제목(AI 번역)**: 사물인터넷 기기를 위한 가벼운 PUF 기반 인증 방식
- 원문 초록 발췌: "We propose a lightweight authentication protocol based on physical

- 핵심 요약(AI 요약):

  2. 칩마다 고유하게 생기는 미세한 차이(PUF)를 "지문"처럼 활용해 가볍게 인증하는 방법 제안
  3. 실험 결과 메모리·전력 사용을 줄이면서도 위조 저항성을 유지함
- 원문 링크: https://arxiv.org/abs/XXXX.XXXXX
```

## 활용 팁
- 키워드는 **영어로 입력**하면 검색되는 논문 수가 훨씬 많음 (학술 논문 대부분이 영어)
- **arXiv**는 물리·전자공학·컴퓨터·AI 최신 논문에 강함 (반도체·보안·머신러닝 동향 파악에 적합)
- **Google Scholar**는 거의 모든 분야 + 피인용 수까지 보여주지만, 자동 접근 시 보안문자로 막힐 때가 잦음
- 같은 키워드로 주기적으로 실행하면 최신 연구 동향을 꾸준히 추적 가능
- 반도체 보안 팀은 `PUF`, `hardware security`, `side-channel attack` 등 키워드로 관련 연구 빠르게 수집 가능
